# ФОЛИО WinСклад --- Бизнес-процесс «Счёт»

## Статус исследования

Версия: 0.1 (экспериментально подтверждено)

## Цель

Исследовать создание учитываемого счета для последующей реализации
собственного API.

## Основные таблицы

  Таблица    Назначение         Статус
  ---------- ------------------ --------
  SCL_NAKL   Шапка документа    ✅
  SCL_MOVE   Строки документа   ✅
  SCL_ARTC   Остатки товара     ✅
  SCL_ADDN   Доп. реквизиты     ⏳
  SCL_PLAT   Платежи            ⏳
  SCL_PMOV   Строки платежей    ⏳

## Подтвержденные факты

### 1. Структура счета

SCL_NAKL хранит шапку документа.

SCL_MOVE хранит строки документа.

Связь:

    SCL_NAKL.UNICUM_NUM
            │
            ▼
    SCL_MOVE.UNICUM_NUM

### 2. Остатки

    KON_KOLCH — фактический остаток
    REZ_KOLCH — свободный остаток

Учитываемый счет:

-   НЕ изменяет KON_KOLCH
-   уменьшает REZ_KOLCH

### 3. Подтвержденные эксперименты

Создание счета: - резерв уменьшается на количество товара.

Изменение количества: - резерв изменяется на дельту.

Удаление строки: - резерв полностью возвращается.

Второй счет: - резерв суммируется.

### 4. Минимальная модель

    INSERT SCL_NAKL
            ↓
    INSERT SCL_MOVE
            ↓
    UPDATE SCL_ARTC.REZ_KOLCH
