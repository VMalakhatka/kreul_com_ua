# ACCOUNT WRITE MAPPING

Статус: предварительный mapping для Java API `/admin/folio/accounts`.

Важно: исходный снимок `UNICUM_NUM = 752327` не найден, но 2026-07-22 добавлен рабочий Excel-снимок `/Volumes/BackUp/Nakl_field.xls` со сравнением:

- `753529` — видимый счёт ФОЛИО;
- `753538` — счёт, созданный API, но неполный и не отображающийся в реестре;
- `753546` — новый ручной счёт ФОЛИО с заполненными реквизитами.

Поэтому ниже разделены:

- поля, подтверждённые текущими docs/экспериментами;
- поля, подтверждённые Excel-сравнением как обязательные/нужные для API;
- поля, которые всё ещё нельзя заполнять “по догадке”.

## Важное про коды типов документов

Для ФОЛИО коды типов документов — кириллические символы:

| Код | Unicode | Значение |
|---|---|---|
| `Р` | `U+0420` | расходная накладная |
| `П` | `U+041F` | приходная накладная |
| `С` | `U+0421` | счёт |

Не использовать латинские `P`/`C`. В SQL-результатах они выглядят похоже, но ФОЛИО UI различает их. Для счёта нужно `TYPE_DOC = 'С'` и `TYPDOCM_PR = 'С'`.

## UNKNOWN / REQUIRES CONFIRMATION

Нельзя заполнять случайными значениями до подтверждения правила:

- обязательные `NOT NULL` поля `SCL_NAKL`, если они есть в конкретной базе и не перечислены выше;
- обязательные `NOT NULL` поля `SCL_MOVE`, если они есть в конкретной базе и не перечислены выше;
- правило расчёта `SUM_VALUT`, `VALUT_CENA`, валютного курса и `COD_VALUT`;
- точное правило для `SUM_ROZN`: Excel-комментарий говорит считать розничную сумму по справочнику товаров;
- поля партии, срока, налогов и служебных идентификаторов строки;
- связь с партнёром/контрагентом `partnerId`;
- необходимость и обязательный набор полей `SCL_ADDN` для видимости/открытия счёта в ФОЛИО UI;
- штатный алгоритм генерации `UNICUM_NUM`, если он есть в ФОЛИО;
- поведение удаления всего счёта: физическое удаление, архивы `SCL_ARCN/SCL_ARCM/SCL_ARCR`, статусы, возврат резерва.

## Контрольная точка перед production write

Перед включением записи в реальную базу нужно получить:

1. `SELECT * FROM SCL_NAKL WHERE UNICUM_NUM = 752327`;
2. `SELECT * FROM SCL_MOVE WHERE UNICUM_NUM = 752327 ORDER BY NUM_PREDMT, RECNO`;
3. `SELECT * FROM SCL_ADDN WHERE UNICUM_NUM = 752327`;
4. схему `NOT NULL`/default для `SCL_NAKL` и `SCL_MOVE`.

После этого mapping нужно обновить и только затем расширять `INSERT`.
